# Notebooks

Diretorio para experimentos, analise exploratoria e validacao visual de resultados.

## Objetivo academico

Os notebooks devem complementar (nao substituir) o pipeline em `src/`, servindo para:

- exploracao de hipoteses;
- analises descritivas;
- visualizacoes para relatorios;
- validacoes rapidas de abordagem.

## Convencao de nomenclatura

- `01_exploracao_inicial.ipynb`
- `02_preprocessamento.ipynb`
- `03_modelagem.ipynb`
- `04_avaliacao.ipynb`

## Boas praticas

1. Incluir celula inicial com objetivo, autor e data.
2. Evitar caminhos absolutos locais.
3. Salvar outputs relevantes no diretorio `reports/` quando forem usados em entrega.
4. Documentar parametros usados para reproducao dos resultados.
